## Rule 1: Use Git in Every Project

## Rule 3: Use GitHub Actively

- Push frequently
- Write meaningful commit messages
- Use pull requests (even solo)
- Review your own diffs
- Use tags for versions

--------------------------------------------

## Rule 4: Simulate Team Environment

Even if working alone:
- Create multiple branches
- Merge in different orders
- Use force-with-lease carefully
- Resolve conflicts

--------------------------------------------

## Rule 5: Practice History Hygiene

Before pushing:
- Squash unnecessary commits
- Rebase cleanly
- Remove debug commits
- Write proper commit messages

Clean history = professional workflow.

--------------------------------------------

## Rule 6: Build Recovery Confidence

At least once per week:
- Reset hard
- Delete branch
- Recover using reflog

Fear disappears when recovery is practiced.
