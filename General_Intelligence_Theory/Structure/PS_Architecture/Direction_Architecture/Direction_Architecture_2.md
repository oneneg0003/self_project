==================================================
OBJECT-LEVEL THINKING MODES
==================================================

--------------------------------------------------
Lateral Thinking [O]

1. Frame-Shifting  
Change representation → re-evaluate structure under new frame.

2. Freedom-Driven  
Postpone constraint filtering → generate diverse alternatives.

3. Counterfactual  
Alter assumption → observe hypothetical outcome pathways.

4. Periphery-First  
Start reasoning from edge/atypical elements → derive new insights.

5. Cross-Linking  
Jump across disconnected concepts → create non-obvious connections.

--------------------------------------------------
Divergent Thinking [O]

1. Breadth-First  
Generate many shallow alternatives → before deep refinement.

2. Hypothesis-First  
Propose multiple candidate models → defer validation until later.

3. Opportunity-First  
Seek high-value potential branches → expand high-promise areas.

4. Category-Crossing  
Mix categories → create hybrid options.

--------------------------------------------------
Critical Thinking [O]

1. Constraint-Driven  
Apply logical constraints → eliminate invalid claims.

2. Evidence-First  
Demand evidence before acceptance → build support incrementally.

3. Causal-Reverse  
Trace conclusion → back to underlying assumptions.

4. Frame-Preserving  
Reason within representation → check internal coherence.

5. Error-Signal Emphasis  
Focus on contradictions → prioritize resolving them.

--------------------------------------------------
Convergent Thinking [O]

1. Best-First  
Select strongest option → analyze it thoroughly.

2. Constraint-Pruning  
Apply criteria → shrink candidate set sharply.

3. Risk-First  
Evaluate negative outcomes → prioritize reliable options.

4. Backward  
Start from target criteria → derive acceptable options.

5. Depth-First (after elimination)  
Follow best candidate deeply → refine solution.

--------------------------------------------------
Analytical Thinking [O]

1. Top-Down  
Overall structure → decompose into precise parts.

2. Bottom-Up  
From parts → derive general structure or composite behavior.

3. Backward  
Goal condition → determine prior logical conditions.

4. Forward  
Premises → derive direct consequences.

5. Causal-Forward  
Identify cause → trace forward consequences logically.

--------------------------------------------------
Integration Thinking [O]

1. Bottom-Up  
Aggregate component insights → build comprehensive structure.

2. Center-First  
Identify core concept → orient other elements around it.

3. Bidirectional Reconciliation  
Adjust parts ↔ whole → achieve consistency.

4. Constraint Harmonization  
Identify conflicting parts → modify until constraints satisfied.

5. Interface-Focused  
Check interactions → ensure components fit collaboratively.

--------------------------------------------------
Structured Thinking [O]

1. Top-Down  
Define scaffold → fill details layer by layer.

2. Level-By-Level  
Complete one abstraction layer → only then proceed.

3. Constraint-Driven  
Enforce structural rules → preserve architecture integrity.

4. Frame-Preserving  
Reason within defined representation → avoid ambiguity.

--------------------------------------------------
Abstraction (Compression) [O]

1. Abstraction-Up  
Abstract out common patterns → form general rules.

2. Bottom-Up  
Aggregate instances → infer higher-level structure.

3. Frame-Preserving  
Retain core relations → remove peripheral detail.

4. Representational Summarization  
Summarize multiple cases → compress into an abstract model.

--------------------------------------------------
Probabilistic Thinking [O]

1. Hypothesis-First  
Propose model → update by evidence.

2. Evidence-First  
Collect data → drive model structure.

3. Belief-Revision  
Adjust confidence in light of new information.

4. Risk-First  
Evaluate downside likelihood → guide decision.

5. Counterfactual  
Estimate how outcomes shift if assumptions change.

--------------------------------------------------
Analogous Thinking [O]

1. Abstraction-Up  
Extract relational patterns → irrespective of domain.

2. Frame-Shifting  
Map structural pattern → to new context.

3. Instantiation-Down  
Apply pattern to specific target situation.

4. Bidirectional Validation  
Check mapping both ways → refine alignment.

5. Structural Mapping  
Align relations not features → emphasis on structure.

==================================================
DUAL-LEVEL THINKING MODES
==================================================

--------------------------------------------------
Strategic Thinking [D]

1. Backward  
Define desired outcome → determine key conditions.

2. Multi-Frame  
Reason from multiple viewpoints → compare consequences.

3. Risk-First  
Evaluate threats → protect critical positions.

4. Opportunity-First  
Identify leverage → maximize gains.

5. Dynamic Progression  
Simulate evolving states → adjust plan over time.

6. Constraint-Driven  
Enforce interaction rules → preserve logical coherence.

--------------------------------------------------
Perspective Thinking [D]

1. Other-Frame  
Adopt alternative viewpoint → reinterpret structure.

2. Self-Frame  
Return to own frame → contrast interpretations.

3. Multi-Frame  
Hold simultaneous perspectives → triangulate insight.

4. Frame-Shifting  
Switch interpretive lens → alternate meaning.

5. Abstraction-Up  
Extract high-level patterns → compare across frames.

--------------------------------------------------
Reinforcement Thinking [D]

1. Forward  
Action → observe reward consequences.

2. Backward  
Propagate reward signal → adjust policy.

3. Perturbation-Based  
Small change → test effect → evaluate update.

4. Opportunity-First  
Prioritize actions with high expected return.

5. Dynamic Progression  
Adjust strategy over evolving state transitions.

--------------------------------------------------
Reflective Thinking [D]

1. Causal-Reverse  
Outcome → trace decision logic backward.

2. Self-Comparison  
Compare action vs intention → detect deviation.

3. Evidence-First  
Review observations → revise internal belief.

4. Abstraction-Up  
Extract lesson → form general insight.

5. Bidirectional Refinement  
Iterate between hindsight insight ↔ improved model.

--------------------------------------------------
Meta-Cognitive Thinking [D]

1. Frame-Monitoring  
Observe current cognitive frame → evaluate fit.

2. Constraint-Driven  
Detect bias/violation → enforce correction.

3. Bidirectional Control  
Switch mode based on feedback signals.

4. Dynamic Adjustment  
Adapt strategy mid-execution when needed.

5. Strategy Metacontrol  
Select reasoning direction → based on context.

==================================================
CONTROL-LEVEL THINKING MODE
==================================================

--------------------------------------------------
Intuitive Thinking [C]

1. Best-First  
Match strongest pattern → generate fast inference.

2. Pattern-Activation  
Retrieve familiar structure → respond quickly.

3. Snapshot Evaluation  
Assess structure briefly → without deep traversal.

4. Implicit Hypothesis  
Generate quick guess → refine if required.

5. Heuristic-Driven  
Use learned shortcuts → reduce search cost.
