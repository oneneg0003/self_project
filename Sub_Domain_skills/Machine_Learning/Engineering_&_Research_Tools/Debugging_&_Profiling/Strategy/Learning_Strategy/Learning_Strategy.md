### Phase 1: Foundation (Understand)
Goal: Build correct mental model (WHY + WHAT)

- Learn definitions and core concepts
- Focus on the problem the tool solves
- Avoid tool overload
- Build structured notes
- Ask: "What layer is this? What does it control?"

- Concept map
- Minimal working examples

### Phase 2: Controlled Practice (Apply)
Goal: Build fluency (HOW)

- Solve small targeted problems
- Repeat core patterns
- Make mistakes intentionally
- Debug your own code

Output:
- Can use without constantly checking docs
- Faster execution
- Less confusion

---

### Phase 3: Stress Testing (Strengthen)
Goal: Break your understanding to deepen it

- Try edge cases
- Use larger inputs/data
- Trigger performance/memory issues
- Recover from errors

Output:
- Robust skill
- Recovery ability
- Confidence under pressure

---

### Phase 4: System Integration (Engineer Level)
Goal: Connect everything into one workflow

- Combine tools into a pipeline
  (Git + ML + debugging + tracking + Docker + deploy)
- Automate repetitive steps
- Improve reproducibility

Output:
- Production-level thinking
- End-to-end skill
