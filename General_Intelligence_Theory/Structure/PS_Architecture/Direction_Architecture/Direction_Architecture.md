# DIRECTIONAL ARCHITECTURE
(Traversal Directions Catalog)

Definition:
Direction specifies how reasoning/search traverses an organization form.
Direction is defined relative to organization.

--------------------------------------------------
Directions for Hierarchy / Tree:
- Top-down
- Bottom-up
- Level-by-level
- Root-to-leaf
- Leaf-to-root
- Core-to-periphery
- Periphery-to-core

--------------------------------------------------
Directions for Sequence / Chain:
- Forward
- Backward
- Sliding-window (local)
- Jump/skip
- Split-merge

--------------------------------------------------
Directions for Graph / Network:
- Depth-first
- Breadth-first
- Hub-first
- Path-focused
- Neighborhood expansion
- Bridge-crossing
- Cycle-aware traversal

--------------------------------------------------
Directions for State Space / Transition System:
- Forward expansion
- Backward chaining
- Frontier expansion
- Heuristic-guided expansion
- Constraint-pruned expansion

--------------------------------------------------
Directions for Set / Unordered Collection:
- Exhaustive scan
- Filter-first
- Sample-then-refine
- Rank-then-scan

--------------------------------------------------
Directions for Matrix / Multi-Dimensional Grid:
- Row-wise
- Column-wise
- Axis-by-axis
- Slice/projection
- Cross-cut traversal (diagonal / multi-axis)

--------------------------------------------------
Directions for Cycle / Feedback Loop:
- Iterative loop traversal
- Perturbation propagation
- Stability-seeking traversal
- Drift tracking across cycles

--------------------------------------------------
Directions for Modular / Clustered System:
- Module-isolated traversal
- Interface-focused traversal
- Incremental integration traversal
- Core module first / peripheral modules later

--------------------------------------------------
Directions for Layered / Multi-Level Stack:
- Upward abstraction (concrete → abstract)
- Downward instantiation (abstract → concrete)
- Cross-layer alignment traversal
- Layer-by-layer traversal
